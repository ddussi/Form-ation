브라우저 개발자 도구 콘솔에서 다음 명령을 사용하세요:

  • formManager.getDetectedForms() - 감지된 폼 목록 확인
  • formManager.manualCheck() - 수동으로 저장 체크 실행
  • formManager.manualSaveTest() - 저장 모달 테스트
  • formManager.manualAutofillTest() - 자동입력 모달 테스트 (4단계 신규)
  • formManager.getStorageDebugInfo() - 저장된 데이터 확인

🎉 5단계 완성! UI/UX 개선 테스트

🍞 토스트 알림 테스트:

  1. 폼에 값을 입력하고 제출
  2. [저장] 클릭 시 "폼 데이터 저장됨 (N개 필드)" 토스트 확인
  3. 새로고침 후 [입력] 클릭 시 "자동입력 완료 (N개 필드)" 토스트 확인
  4. 토스트는 우측 상단에 3초간 표시됨

⚙️ 관리 페이지 테스트:

  1. 크롬 확장 아이콘 우클릭 → "옵션" 클릭
  2. 저장된 폼 데이터 목록 확인
  3. 각 폼의 저장/자동입력 모드 변경 테스트
  4. 개별 삭제, 사이트별 삭제, 전체 삭제 테스트
  5. 통계 정보(저장된 폼 수, 사용 용량) 확인

🔄 다중 폼 자동입력 테스트:

  1. 위에서 저장한 후 페이지를 새로고침
  2. 페이지 로드 시 순차적으로 자동입력 모달 표시
  3. 첫 번째 폼 모달에서 "⏭️ 대기 중인 폼: N개" 확인
  4. 각 모달에서 [입력] 클릭하면 해당 폼만 채워짐
  5. 토스트 알림으로 자동입력 완료 피드백

🎯 수동 테스트:

  • formManager.manualSaveTest() - 저장 모달 즉시 표시
  • formManager.manualAutofillTest() - 자동입력 모달 즉시 표시

🔔 브라우저 알림 시스템 테스트 (NEW!):

  1. 폼에 값을 입력하고 제출 버튼 클릭
  2. 페이지가 이동해도 시스템 알림이 표시됨
  3. 알림에서 "💾 저장", "❌ 이번에는 안함", "🚫 다시 묻지 않음" 버튼 확인
  4. 다른 탭이나 앱을 사용 중에도 알림 확인 가능
  5. 새로고침 후 자동입력 알림도 동일하게 시스템 레벨에서 표시
